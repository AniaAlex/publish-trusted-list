# go-trust-ecosystem-602-lote

Generates a List of Trusted Entities (LoTE) per ETSI TS 119 602, using the
[`g119612`](../g119612) Go toolkit.

---

## Usage

```bash
# 1. Build tsl-tool (once)
make -C ../g119612 tsl-tool

# 2. Generate test certificates for each entity (once)
make keys

# 3. Generate the LoTE
make lote
```

Output is written to `output/lote-SE.json`.

---

## Directory structure

```
go-trust-ecosystem-602-lote/
├── scheme.yaml             # LoTE scheme metadata (operator, territory, sequence number)
├── pipeline.yaml           # tsl-tool pipeline: generate-lote → publish-lote
├── Makefile
├── entities/               # One subdirectory per trusted entity
│   ├── italy-pid/
│   │   ├── entity.yaml     # Entity metadata (name, type, status, services)
│   │   └── test_cert.pem   # X.509 certificate (generated by make keys)
│   ├── germany-pid/
│   ├── italy-wallet/
│   ├── france-wallet/
│   └── eu-wrpac/
└── output/                 # Generated LoTE JSON (gitignored)
```

### `scheme.yaml` fields

| Field | Description |
|---|---|
| `operatorNames` | Multilingual name of the scheme operator |
| `schemeName` | Multilingual name of the scheme |
| `schemeType` | ETSI URI identifying the scheme type |
| `territory` | Two-letter territory code (used in output filename) |
| `sequenceNumber` | Incremented on each publication |

### `entity.yaml` fields

| Field | Description |
|---|---|
| `names` | Multilingual entity name |
| `entityId` | Unique identifier for the entity |
| `entityType` | ETSI URI for the entity type |
| `status` | `granted` or `withdrawn` |
| `services` | List of services (name, type, status) |

### Digital identity files

Place one or more of the following in each entity directory:

| Extension | Type |
|---|---|
| `.pem` / `.crt` / `.cer` | X.509 certificate (PEM or DER) |
| `.jwk` | JSON Web Key |
| `.did` | Decentralised Identifier (must start with `did:`) |

---

## Pipeline steps used

| Step | Purpose |
|---|---|
| `generate-lote` | Reads `scheme.yaml` + `entities/` → builds LoTE in memory |
| `publish-lote` | Writes `output/lote-<territory>.json` (unsigned) |

To publish a JWS-signed LoTE, extend `pipeline.yaml`:

```yaml
- generate-lote:
    - .

- publish-lote:
    - ./output
    - /path/to/cert.pem
    - /path/to/key.pem
```
