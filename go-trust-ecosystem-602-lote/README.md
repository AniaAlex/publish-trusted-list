# go-trust-ecosystem-602-lote

Generates a List of Trusted Entities (LoTE) per ETSI TS 119 602, using the
[`g119612`](../g119612) Go toolkit.

---

## Usage

```bash
# 1. Build tsl-tool (once)
make -C ../g119612 tsl-tool

# 2. Generate test certificates for each entity (once)
make keys

# 3. Generate the LoTE (unsigned)
make lote

# 4. Generate and JWS-sign the LoTE
make sign
```

Output is written to `output/lote-SE.json` (unsigned) and `output/lote-SE.json.jws` (signed).

## Configuration

### `scheme.yaml`

| Field | Description |
|---|---|
| `operatorNames` | Multilingual name of the scheme operator |
| `schemeName` | Multilingual name of the scheme |
| `schemeType` | ETSI URI identifying the scheme type |
| `territory` | Two-letter territory code (used in output filename) |
| `sequenceNumber` | Incremented on each publication |

### `entity.yaml`

| Field | Description |
|---|---|
| `entityId` | Unique URI identifying the entity |
| `names` | Multilingual entity name |
| `entityType` | Role of the entity (e.g. `pid-provider`, `credential-issuer`, `verifier`) |
| `status` | Trust status URI — `granted` or `withdrawn` |
| `services` | List of services, each with `serviceType`, `serviceNames`, `status` |

### Digital identity files

Place one or more of the following in each entity directory:

| Extension | Identity type |
|---|---|
| `.pem` / `.crt` / `.cer` | X.509 certificate (PEM or DER) |
| `.jwk` | JSON Web Key |
| `.did` | Decentralised Identifier (must start with `did:`) |

All files found are included as `digitalIdentities` entries for that entity.

## Entities

| Directory | Entity | `entityType` | `serviceType` |
|---|---|---|---|
| `example-ca` | Example CA | — | `Svctype/CA/QC` |
| `example-issuer` | Example Credential Issuer | `credential-issuer` | `Svctype/CA/QC` |
| `example-verifier` | Example Verifier | `verifier` | — |
| `se-pid-issuer-a` | Swedish PID Issuer A | `pid-provider` | `Svctype/EAA/PID` |
| `se-pid-issuer-b` | Swedish PID Issuer B | `pid-provider` | `Svctype/EAA/PID` |

Entities with the same `entityType` are distinguished by their `entityId` URI.
To find all PID providers a consumer filters `trustedEntities` where `entityType == "pid-provider"`.

## Pipeline steps

| Step | Purpose |
|---|---|
| `generate-lote` | Reads `scheme.yaml` + `entities/` → builds LoTE in memory |
| `publish-lote` | Writes `output/lote-<territory>.json`; optionally JWS-signs if cert/key are passed |

`pipeline.yaml` runs unsigned output. `pipeline-sign.yaml` adds `lote-signing-cert.pem` /
`lote-signing-key.pem` to the `publish-lote` step — generated by `make sign-keys`.
